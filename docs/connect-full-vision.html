Executive Summary

Imotara currently provides AI/LLM-based emotional guidance and companionship. This document describes Imotara Connect — a real-human consultancy layer that allows verified human consultants to provide emotional, relational, and wellness support to users across all languages and time zones.

This feature positions Imotara beyond a chatbot into a full mental wellness ecosystem: AI-first for everyday reflection, and human-first for deeper support moments. Both layers share the same user account, history, and settings.

Billing Model (Updated from original spec): Per-minute pre-pay wallet system. Users recharge based on expected duration. Imotara takes 20% at recharge time. Both user and consultant see minutes (not money). Session auto-ends when balance reaches zero. Unused minutes stay for next session with the same consultant.

User Flows

3.1 User (Seeker) Journey

Browse Consultants
  → Filter: category / language / rating / availability / gender / expertise
  → View profile: photo, bio, expertise tags, languages, rate, reviews
  → Instant connect (consultant online now)
  │   → Check wallet balance (sufficient for ≥ 1 minute?)
  │   → If insufficient: Recharge flow
  │       → Select duration (15 / 30 / 60 / custom min)
  │       → See breakdown: consultant fee + 20% platform fee = total
  │       → Pay via Razorpay / Stripe
  │       → Balance credited (in minutes)
  │   → Send instant request → Consultant accepts (60s window)
  │   → Enter session immediately (text chat MVP, audio/video Phase 3)
  └── Schedule meeting (consultant offline)
      → Type preferred time in text field
      → Submit request → Consultant sees it and accepts/proposes alternative
      → When consultant accepts: recharge to confirm
      → Join at agreed time
→ Post-session: Rate (1–5 stars) + optional review text
→ Optional: Rebook, AI reflection note linked to session

3.2 Consultant (Provider) Journey

Register as Consultant (via same Imotara account)
  → Fill profile: name, photo, expertise tags, bio, languages, rate, currency
  → Accept Code of Conduct (mandatory checkbox)
  → Submit → Admin review (email + in-app status)
  → Approved → Set online status → Go Live

Accept requests:
  ├── Instant: notification arrives → 60s to Accept / Decline
  │   → On accept: session starts immediately
  └── Scheduled: approve / suggest alternative time

Conduct session (text chat MVP):
  → Both see live minute countdown
  → Session auto-ends at 0 balance
  → Add private notes (not visible to user)

After session:
  → Earnings credited to wallet (80% of session amount)
  → Request payout (UPI / Bank / PayPal)

Revenue Model & Billing

5.1 Per-Minute Billing (Core Model)

Key design principle: Both user and consultant see time (minutes), never money amounts. The 20% platform fee is invisible — deducted at recharge. This eliminates awkward "how much did I spend?" conversations during sessions.

Consultant sets rate: R per minute (their currency)
User recharges: A (any supported currency)
Platform fee (20%): A × 0.20 → Imotara
Consultant credit (80%): A × 0.80
Minutes credited: (A × 0.80) / R
During session: 1 min deducted every 60 seconds
Warning at: <1 minute remaining
Auto-complete at: 0 minutes
After session (T min used):
Imotara keeps: already held from recharge
Consultant earns: T × R × 0.80

Safety, Compliance & Trust

10.1 Non-Clinical Disclaimer (Mandatory — MVP)

Display on: Every consultant card, consultant profile page, session start screen, consultant registration form.

"Imotara Connect provides peer wellness companionship only. Companions are not licensed therapists or medical professionals. If you are in crisis, please use the help button below."

10.2 Emergency Protocol (Mandatory — MVP)

Single-tap "Need Help?" button inside every active session (always visible, even during text chat). Opens a modal — does not navigate away from session.

10.3 Age Gate (Mandatory — MVP)

Under-18 users cannot access /connect. Check profiles.date_of_birth on page load. If null, prompt for birth year. Redirect under-18 to /connect/age-restricted page explaining the restriction.

10.4 Code of Conduct (Mandatory — MVP)

Consultants must check a CoC box at registration. Stored as coc_agreed = true. API rejects registration without it.

CoC prohibitions: No clinical / medical / legal advice · No romantic or sexual advances · No financial solicitation outside the platform
Enforcement: Warning → 30-day suspension → Permanent ban (three-strike policy in Phase 2)

10.6 GDPR & DPDPA Compliance

  • →Session content not stored beyond what's needed for history
  • →Government ID docs in encrypted private Supabase bucket (Phase 2) — signed URLs only for admin reviewers
  • →DPIA (Data Protection Impact Assessment) required before Connect goes live
  • →Consultant right-to-erasure: profile + messages deleted on account deletion (cascade)
  • →Retention: financial records (7 years), session content (user-controlled deletion)
