MVP Scope

What this plan covers: A text-chat based mental wellness companion marketplace with per-minute billing, multi-currency support, and full Supabase Realtime integration. Audio/video calls, additional consultant categories, and advanced scheduling are deferred to later phases.

In MVP

  • Mental Wellness Companion (male + female)
  • Text chat via Supabase Realtime
  • Per-minute pre-pay wallet billing
  • 20% platform fee (invisible to users)
  • Multi-currency rates (7 currencies)
  • Consultant profile: photo, bio, expertise tags
  • Online status + instant connect
  • Offline → request scheduled meeting
  • Session history with chat replay
  • Post-session ratings + reviews
  • Admin approval + email notification
  • Non-clinical disclaimer
  • Emergency crisis button
  • Age gate (under-18 blocked)
  • CoC agreement at registration

Deferred (Later Phases)

  • Audio / video calls (Daily.co)
  • 11 other consultant categories
  • Calendar slot picker + timezone
  • Government ID verification
  • Featured / paid listings

Per-Minute Billing Model

Core principle: User pays upfront. Both parties see minutes (not money). Imotara's 20% is invisible — taken at recharge time. Unused minutes stay in wallet for next session with the same consultant.

Calculation

Consultant sets rate: R per minute (their currency)

User recharges: A (their currency)
Platform fee (20%): A × 0.20 → Imotara holds
Usable credit (80%): A × 0.80
Minutes credited: (A × 0.80) / R
Both user & consultant see: X minutes (same number)
Neither sees: the 20/80 split

During session (every 60s):
Deduct 1 min from wallet
At <1 min remaining: warning banner shown
At 0 min: session auto-completes
After session (T minutes used):
Imotara keeps: already held from recharge
Consultant credited: T × R × 0.80
Unused balance: stays in user wallet (consultant-specific)

Example

Consultant rate: ₹50 / minute
User recharges: ₹1,800
Platform fee: ₹1,800 × 0.20 = ₹360
Usable credit: ₹1,800 × 0.80 = ₹1,440
Minutes credited: ₹1,440 ÷ ₹50 = 28.8 minutes
After 28.8 min session:
Imotara receives: ₹360 (already held)
Consultant earns: ₹1,440
Unused balance: ₹0 (session fully consumed)

Recharge UX (Duration-First)

Step 1: User taps "Talk Now" on consultant profile
Step 2: Insufficient balance check → Recharge screen opens

Step 3: Duration picker shown
  [15 min]  [30 min]  [60 min]  [Custom: ___]

Step 4: Live cost breakdown

  ─────────────────────────────────

Database Schema

6 new tables. All with Row-Level Security. connect_messages and connect_sessions added to Supabase Realtime publication.

connect_consultants

CREATE TABLE connect_consultants (
  id                 uuid        PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id            uuid        NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  display_name       text        NOT NULL,
  gender             text        CHECK (gender IN ('male','female')),
  photo_url          text,
  bio                text        CHECK (char_length(bio) <= 500),
  expertise_tags     text[],     -- ['Stress & Anxiety','Loneliness',...]
  languages          text[],
  rate_per_min       numeric(10,2) NOT NULL,
  currency_code      text        NOT NULL DEFAULT 'INR',
  rate_per_min_inr   numeric(10,2),  -- snapshot for sorting
  availability_note  text,
  coc_agreed         boolean     NOT NULL DEFAULT false,
  status             text        NOT NULL DEFAULT 'pending'
                     CHECK (status IN ('pending','approved','suspended','rejected')),
  rejection_reason   text,
  is_online          boolean     NOT NULL DEFAULT false,
  rating_avg         numeric(3,2) DEFAULT 0,
  rating_count       integer     DEFAULT 0,
  sessions_completed integer     DEFAULT 0,
  created_at         timestamptz DEFAULT now(),
  updated_at         timestamptz DEFAULT now()
);

connect_sessions

CREATE TABLE connect_sessions (
  id                  uuid        PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id             uuid        NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  consultant_id       uuid        NOT NULL REFERENCES connect_consultants(id),
  type                text        NOT NULL CHECK (type IN ('instant','scheduled')),
  status              text        NOT NULL DEFAULT 'pending'
                      CHECK (status IN ('pending','accepted','declined','active',
                                        'completed','cancelled')),
  scheduled_note      text,       -- user's preferred time in free text
  started_at          timestamptz,
  ended_at            timestamptz,
  minutes_used        numeric(10,2) DEFAULT 0,
  amount_charged      numeric(12,2) DEFAULT 0,
  currency_code       text        DEFAULT 'INR',
  rating              integer     CHECK (rating BETWEEN 1 AND 5),
  review_text         text        CHECK (char_length(review_text) <= 200),
  review_submitted_at timestamptz,
  created_at          timestamptz DEFAULT now()
);

connect_messages

CREATE TABLE connect_messages (
  id          uuid  PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id  uuid  NOT NULL REFERENCES connect_sessions(id) ON DELETE CASCADE,
  sender_id   uuid  NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  content     text  NOT NULL CHECK (char_length(content) <= 2000),
  created_at  timestamptz DEFAULT now()
);
-- Enable Realtime
ALTER PUBLICATION supabase_realtime ADD TABLE connect_messages;
ALTER PUBLICATION supabase_realtime ADD TABLE connect_sessions;

connect_wallet

CREATE TABLE connect_wallet (
  user_id          uuid  PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  balance_amount   numeric(12,2) NOT NULL DEFAULT 0,
  balance_currency text          NOT NULL DEFAULT 'INR',
  earned_amount    numeric(12,2) NOT NULL DEFAULT 0,   -- for consultants
  earned_currency  text          NOT NULL DEFAULT 'INR',
  pending_payout   numeric(12,2) NOT NULL DEFAULT 0,
  updated_at       timestamptz DEFAULT now()
);

connect_recharges

CREATE TABLE connect_recharges (
  id                   uuid  PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id              uuid  NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  consultant_id        uuid  NOT NULL REFERENCES connect_consultants(id),
  razorpay_order_id    text,
  razorpay_payment_id  text,
  amount               numeric(12,2) NOT NULL,
  currency_code        text  NOT NULL DEFAULT 'INR',
  minutes_credited     numeric(10,2) NOT NULL,
  platform_fee         numeric(12,2) NOT NULL,
  consultant_credit    numeric(12,2) NOT NULL,
  status               text  NOT NULL DEFAULT 'pending'
                       CHECK (status IN ('pending','completed','failed')),
  created_at           timestamptz DEFAULT now()
);

connect_payouts

CREATE TABLE connect_payouts (
  id                 uuid  PRIMARY KEY DEFAULT gen_random_uuid(),
  consultant_user_id uuid  NOT NULL REFERENCES auth.users(id),
  amount             numeric(12,2) NOT NULL,
  currency_code      text  NOT NULL DEFAULT 'INR',
  payout_method      text  CHECK (payout_method IN ('upi','bank','paypal')),
  payout_details     jsonb,
  status             text  NOT NULL DEFAULT 'pending'
                     CHECK (status IN ('pending','processing','completed','failed')),
  admin_note         text,
  created_at         timestamptz DEFAULT now(),
  processed_at       timestamptz
);

RLS Note: All writes to connect_wallet go through API routes using the service role. Direct client writes are blocked. connect_consultants status field changes are admin-only via service role.

4
API Routes

Consultant Routes

GET/api/connect/consultantsList approved consultants (filters: gender, expertise_tags, languages, is_online)
GET/api/connect/consultants/[id]Single consultant public profile
POST/api/connect/consultant/registerSubmit consultant registration (any authed user)
PATCH/api/connect/consultant/statusToggle is_online (approved consultants only)
PATCH/api/connect/consultant/profileUpdate bio, tags, rate, photo (cannot change status)
GET/api/connect/consultant/earningsConsultant earnings breakdown + session history
POST/api/connect/consultant/payoutRequest payout → creates connect_payouts row + notifies admin

Session Routes

POST/api/connect/sessionsCreate session (instant or scheduled). type=instant notifies consultant via Realtime
GET/api/connect/sessionsUser's session list (most recent first)
PATCH/api/connect/sessions/[id]Accept / decline / complete / cancel (participants only)
POST/api/connect/sessions/[id]/tickDeduct 1 minute every 60s. Auto-completes at 0. Returns {remaining_minutes, status}
POST/api/connect/sessions/[id]/reviewSubmit rating + review after session completes. Updates consultant rating_avg

Wallet Routes

GET/api/connect/walletGet balance amount + minutes remaining + earned amount (if consultant)
POST/api/connect/wallet/recharge/createCreate Razorpay order. Body: {consultant_id, amount, currency_code}
POST/api/connect/wallet/recharge/verifyVerify HMAC signature → credit wallet. Splits 20/80 in connect_recharges

Admin Routes

GET/api/admin/connect/pendingPending consultant applications list
PATCH/api/admin/connect/[id]/approveApprove or reject. Sends email to consultant
GET/api/admin/connect/consultantsAll consultants (all statuses)
PATCH/api/admin/connect/consultants/[id]Suspend / reinstate

// In ConnectChatScreen.tsx
const channel = supabase
  .channel(`connect:session:${sessionId}`)
  .on('postgres_changes', {
    event: 'INSERT',
    schema: 'public',
    table: 'connect_messages',
    filter: `session_id=eq.${sessionId}`
  }, (payload) => {
    setMessages(prev => [...prev, payload.new]);
  })
  .subscribe();

// Also subscribe to session status changes
const statusChannel = supabase
  .channel(`session:status:${sessionId}`)
  .on('postgres_changes', {
    event: 'UPDATE',
    schema: 'public',
    table: 'connect_sessions',
    filter: `id=eq.${sessionId}`
  }, (payload) => {
    setSessionStatus(payload.new.status);
  })
  .subscribe();

Safety & Compliance

1. Non-Clinical Disclaimer

Display text: "Imotara Connect provides peer wellness companionship only. Companions are not licensed therapists or medical professionals. If you are in crisis, please use the help button below."

Shown on: consultant card (as tooltip/footer), consultant profile page (banner at top), session start screen (before first message), consultant registration form.

2. Emergency Crisis Button & Hotlines

  IN: [

  ],
  US: [{ name: '988 Suicide & Crisis Lifeline', number: '988'   }],
  GB: [{ name: 'Samaritans',           number: '116 123'        }],

};

Button location: Inside ConnectChatScreen header (red icon, always visible). On web: fixed bottom-right button on /connect/session/[id]. Tapping opens a modal — no navigation away from session.

3. Age Gate

// src/app/connect/page.tsx — server component check
const profile = await getUserProfile(userId);
const birthYear = new Date(profile.date_of_birth).getFullYear();
const age = new Date().getFullYear() - birthYear;
if (age < 18) redirect('/connect/age-restricted');

If date_of_birth is null, show a prompt to enter birth year before proceeding. Do not allow bypass.

4. Code of Conduct Agreement

CoC text at registration: "I agree to Imotara's Code of Conduct for Wellness Companions: I will not provide clinical, medical, or legal advice; will not make romantic or sexual advances toward users; will not solicit money outside the Imotara platform. I understand that violations result in immediate account suspension."

Stored as coc_agreed = true on the connect_consultants row. Registration API rejects submission if coc_agreed !== true.
