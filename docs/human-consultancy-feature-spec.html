Executive Summary

Imotara currently provides AI/LLM-based emotional guidance and companionship. This document proposes Imotara Connect — a real-human consultancy layer that allows verified human consultants to provide audio and video-based emotional, relational, and wellness support to users across all languages and time zones.

This feature positions Imotara beyond a chatbot into a full mental wellness ecosystem: AI-first for everyday reflection, and human-first for deeper support moments. Both layers share the same user account, history, and settings.

User Flows

3.1 User (Seeker) Journey

Browse Consultants
  → Filter: category / language / rating / availability / gender
  → View profile: photo, bio, expertise, languages, rates, reviews
  → Choose session type: Audio or Audio+Video
  → Check availability in own time zone
      ├── Instant connect (consultant online now)
      │     → Send request → Consultant accepts (60s window)
      │     → Enter session immediately
      └── Schedule appointment
            → Pick slot → Pay → Confirmation email + push
            → Reminder (24h, 1h, 10min before)
            → Join at scheduled time
  → Post-session: Rate + review + mood check-in
                → Optional: Rebook, Tip, AI reflection note

3.2 Consultant (Provider) Journey

Register as Consultant
  → Fill profile: name, photo, categories, languages, rates, ID doc
  → Set mode: Audio-only OR Audio+Video
  → Set availability: weekly schedule + time zone
  → Submit → Imotara Verification Review (2–5 business days)
  → Approved → Go Online
      ├── Accept live requests or manage upcoming bookings
      ├── Conduct session via Imotara Call interface
      └── Post-session: Add private notes → Earnings credited (24h)
  → Withdraw earnings: Bank / UPI / PayPal

Safety, Compliance & Trust

9.1 Non-Clinical Disclaimer

Shown prominently on every consultant profile and booking confirmation: "Imotara Connect provides peer support and wellness companionship. Consultants are not licensed therapists or medical professionals. If you are in crisis, please contact a mental health helpline."

9.2 Emergency Protocol

Single-tap "I need urgent help" button during any session. Immediately surfaces country-detected crisis lines:

9.3 Code of Conduct

Consultants agree at registration to strict prohibitions: no romantic or sexual overtures, no clinical or medical advice, no financial solicitation. Violations follow a three-strike policy: warning → 30-day suspension → permanent ban.

9.4 Trust & Safety Metrics (Auto-monitored by Admin)

SignalThresholdAction
Low ratingBelow 3.5 stars on 10+ sessionsAuto-flag for admin review
Session abandonmentConsultant abandonment rate > 20%Flag for investigation
User reports3+ reports within 30 daysTemporary suspension pending review
Emergency signalAny triggerSession logged; admin notified same day

9.5 Privacy & Data Protection

GDPR and DPDPA (India Digital Personal Data Protection Act) compliant. Session content not stored by default. ID documents stored in encrypted private Supabase bucket (AES-256), accessible only to verified admin reviewers via short-lived signed URLs. Retention: 7 years post-deletion for legal compliance.

Multilingual Support

All user-facing text is served through Imotara's existing i18n infrastructure. New additions required:

  • → Consultant bio: AI-assisted translation at registration time, stored as jsonb {lang: "text"}
  • → Booking confirmation emails in user's configured app language
  • → All 12 category names and descriptions translated across all supported languages
  • → "Speaks your language" badge when consultant's language matches user's app language
  • → Crisis hotline numbers localised by detected country

Admin Panel — Gap Analysis & Required Enhancements

11.1 What the Current Admin Panel Covers

The existing /admin panel handles: super-admin login (email/password + legacy secret key), user license management (tier, status, expiry, token balance), organization management (create, edit, member management, license pools), blog comment moderation, and admin action audit trail.

Verdict: The current admin panel covers 0% of Imotara Connect requirements. It must be extended with 5 new tabs and the existing Moderation tab extended. The existing auth system, UI patterns (dark glass cards, table layout, action badges), and API conventions (Bearer token, PATCH with audit trail) are fully reusable — this is additive development, not a rebuild.
